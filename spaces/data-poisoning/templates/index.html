Data Poisoning Lab

RAG corpus poisoning attacks against the NexaCore Knowledge Hub. The full 4-tab SPA ships in Phase 4b.

Backend-only deploy

The frontend SPA (Info / RAG Poisoning / Defenses / Corpus Browser tabs) is authored in Phase 4b. For Phase 1, the API surface is live at:

  • GET /health — token state, embedding/LLM model IDs, attack count, corpus size
  • GET /api/attacks — list of 6 RP attack definitions
  • POST /api/attack — run a canned RAG poisoning attack (canned mode only at v1; defenses form field accepted but not yet enforced)
  • OpenAPI docs at /docs

v1 scope

6 RAG poisoning attacks (RP.1 — RP.6) anchored in the NexaCore Knowledge Hub scenario. Stack: Groq llama-3.3-70b-versatile + sentence-transformers MiniLM-L6 embeddings + in-memory cosine retrieval. Hardware: HF Spaces cpu-basic.

Tracking: issue #22. Specs: spaces/data-poisoning/specs/ in the monorepo.
