🖼️ Multimodal Security Lab

Phase 3 • Backend + 4 defenses Image-based attacks against multimodal LLMs · NexaCore DocReceive scenario

This is a backend-only deploy. The frontend SPA shell ships in Phase 4. For now, the API surface is live at:
  • GET /health — token state, inference provider, model id, attack & image counts
  • GET /api/attacks — list of 12 attack definitions
  • POST /api/attack — run a canned-image attack with optional defenses (defenses form field accepts a JSON subset of ["ocr_prescan", "output_redaction", "boundary_hardening", "confidence_threshold"])
  • OpenAPI docs at /docs

Tracking: issue #15. Specs: spaces/multimodal/specs/ in the monorepo. Defense calibration: docs/phase3-calibration.md.

v1 scope: 6 Image Prompt Injection attacks (P1.x) + 6 OCR Poisoning attacks (P5.x) anchored in NexaCore DocReceive. Hardware: HF Spaces cpu-basic. Inference: hosted Qwen/Qwen2.5-VL-72B-Instruct via HF Inference Providers (OVH cloud) — warm-served, no cold-start; ~10–20s per attack call.
