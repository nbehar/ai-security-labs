# Overview Spec — exam-admin Space

## Goal

Provide instructors with a verified-receipt viewer and LTI 1.3 grade-passback tool that closes the summative assessment loop: student downloads signed artifact → instructor uploads artifact here → HMAC verified → short answers scored → grade posted to Canvas/Moodle.

---

## Inputs

- A signed receipt `.json` file produced by any AI Security Labs exam-mode space (Red Team, Detection & Monitoring, etc.)
- `EXAM_SECRET` environment variable (set as a private HF Space secret — never exposed to client)
- Optional: `EXAM_LTI_LINEITEM_URL`, `EXAM_LTI_CLIENT_ID`, `EXAM_LTI_PLATFORM_URL`, `EXAM_LTI_PRIVATE_KEY_PEM` for LTI 1.3 grade passback

---

## Scope

**In scope (v1):**
- HMAC-SHA256 receipt verification using `verify_receipt()` from `framework/exam_token.py`
- Score breakdown display: practical per-exercise scores, MCQ results, short-answer responses
- Short-answer rubric panel: 5 criteria × 4 pts per question, instructor enters scores
- Single-receipt verification (drag-drop upload)
- Batch verification: upload multiple receipts, get summary table + CSV export
- LTI 1.3 grade passback to Canvas/Moodle via `pylti1p3` (AGS endpoint)
- Rubric API: `GET /api/rubric/{lab_id}` returns rubric criteria sourced from `exam_questions.py`

**Out of scope (v1):**
- LTI deep-link launch (students launching directly from Canvas into the lab)
- Printing/PDF receipt export
- Database persistence of grading sessions (in-memory only; acceptable at course volume)
- OAuth2 login / instructor authentication (EXAM_SECRET acts as access control)

---

## Audience

Instructors administering a graduate-level course using AI Security Labs. They understand JSON but are not expected to run Python scripts. The UI must support the grading workflow without command-line access.

---

## Success Criteria

1. An instructor can upload a receipt, see a green/red HMAC badge, and read the score breakdown without reading documentation.
2. Tampered receipts (any field modified) always produce `hmac_valid: false`.
3. Short-answer rubric criteria are legible and match the criteria in `exam_questions.py`.
4. A grade can be posted to Canvas in ≤ 3 clicks after verification.
5. Batch mode processes 30 receipts in < 5 seconds.
6. No `EXAM_SECRET` value ever appears in any HTTP response body or browser console.

---

## Scenario

During a final exam, 28 students in Section A complete the Red Team lab with `exam_v1` dataset and download their signed receipts. After the exam window closes, Prof. Behar:

1. Opens `nikobehar/ai-sec-lab-exam-admin` (private HF Space)
2. Drags all 28 `.json` files onto the Batch Verify panel
3. The table shows: student_id, practical_score, MCQ_score, HMAC badge (green/red)
4. Clicks any row to expand: reads the student's 3 short answers side-by-side with the rubric
5. Enters per-criterion scores (0–4) for each of the 3 questions
6. Clicks "Submit Grade to Canvas" → grade appears in Canvas Gradebook within 30 seconds
7. Exports CSV for the registrar

## Acceptance Checks

- [ ] `GET /health` returns `{"status":"ok","exam_secret_configured":true}` when `EXAM_SECRET` is set
- [ ] `GET /health` returns `{"status":"ok","exam_secret_configured":false}` when `EXAM_SECRET` is missing (graceful degradation — verify will always return invalid)
- [ ] `POST /api/verify` with a receipt generated by `exam_token.sign_receipt()` returns `hmac_valid: true`
- [ ] `POST /api/verify` with any field mutated returns `hmac_valid: false`
- [ ] `POST /api/verify` with missing `hmac_sha256` field returns `hmac_valid: false`
- [ ] `GET /api/rubric/red-team` returns rubric criteria matching `exam_questions.py` QUESTION_BANKS
- [ ] `GET /api/rubric/detection-monitoring` returns rubric criteria
- [ ] `GET /api/rubric/unknown-lab` returns 404
- [ ] `GET /api/lti/jwks` returns a JSON object with a `keys` array containing at least one RSA key
- [ ] `POST /api/batch-verify` with 5 receipts (3 valid, 2 tampered) returns correct hmac_valid flags
